# Language Mappings, Mappings & Mappings Reference

This guide covers offline grammar parsing, tone transposition guidelines, translation dictionaries, and summary styling.

## 1. Grammatical Structures & Mappings
* **Active Voice vs. Passive Voice**:
  * *Passive*: "The server was restarted by the developer because of memory leaks."
  * *Active*: "The developer restarted the server to resolve memory leaks."
* **Tenses Alignment**: Coordinate timestamps and actions. Match past/present tense across lists to avoid context switching.
* **Parallelism**: Ensure lists start with identical parts of speech (e.g. all verbs, all nouns, all gerunds).

## 2. Text Simplification & Tone Transpositions
* **Formal / Professional Tone**: Employs third-person voice, avoids contractions, uses industry jargon correctly, and highlights business objectives.
* **Casual / Conversation Tone**: Employs first-person, makes use of simple active words, avoids complex jargon, and highlights personal interactions.
* **Academic / Research Tone**: Employs passive voice where necessary to highlight objects of research, cites metrics, and follows formal logic pathways.

## 3. Translation Dictionary Guidelines
* **Word Mappings**: Translating terms requires identifying contextual synonyms (e.g., "bank" can be a financial institution or a river border).
* **Summarization Rules**:
  * Extract top 3-5 core points.
  * Reduce word count by at least 60%.
  * Maintain thematic layout.
